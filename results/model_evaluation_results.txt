1. PROJECT OVERVIEW
───────────────────────────────────────────────────────────────────────────
  Task              : Indian Sign Language (ISL) Gesture Recognition
  Domain            : Banking & Financial Services
  Total Classes     : 78 banking-specific signs
  Input Features    : 126 per frame (21 keypoints × 2 hands × 3 coords)
  Sequence Length   : 30 frames (normalized)
  Feature Extractor : Google MediaPipe Hand Landmarker


2. DATASET SUMMARY
───────────────────────────────────────────────────────────────────────────
  Total Samples     : ~4,200+ gesture sequences
  Training Set      : ~3,360 samples (80%)
  Validation Set    : ~420 samples (10%)
  Test Set          : ~420 samples (10%)
  Augmentation      : 14 variants per sample (noise, stretch, rotation,
                      mirror, scale, frame dropout)
  Data Format       : NumPy .npy arrays (preprocessed landmarks)

6. KEY OBSERVATIONS
───────────────────────────────────────────────────────────────────────────

  • The KNN baseline achieves only 10.20% accuracy on 78 classes, which
    is barely above random chance (1/78 ≈ 1.28%). This demonstrates that
    simple distance-based classification fails on high-dimensional
    temporal gesture data.

  • The CNN-BiLSTM model achieves 99.91% test accuracy with near-perfect
    precision, recall, and F1 across all 78 classes. The model converges
    rapidly (val_acc reaches 100% by epoch 5) and maintains stability.

  • The attention mechanism allows the model to dynamically focus on the
    most discriminative frames within each gesture sequence, which is
    critical for distinguishing visually similar banking signs.

  • Online augmentation (noise injection, temporal stretching, rotation,
    scaling, frame dropout) combined with Mixup prevents overfitting
    despite the relatively small dataset.

  • The model size is only 3.01 MB, making it extremely efficient for
    deployment on Hugging Face Spaces (Docker) and real-time inference.

  • The feedback retraining loop allows continuous model improvement:
    users correct wrong predictions → data saved → model retrained →
    pushed to production automatically.
